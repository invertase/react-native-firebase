# RNFB Expo Harness

This app is a checked-in Expo development-build harness for validating React Native Firebase Expo config plugins, `expo prebuild`, and local Firebase setup.

## What it is for

- validating Expo config plugin behavior for local workspace `@react-native-firebase/*` packages
- comparing workspace packages with published RNFB packages without changing the bare harness
- reusing local `GoogleService-Info.plist` and `google-services.json` files without committing them
- exercising `expo prebuild`, `expo run:ios`, and `expo run:android` against the repo's Expo integration path

## Quick start

From the repo root:

```bash
yarn app:expo:sync
yarn app:expo:start
yarn app:ios:expo
# or
yarn app:android:expo
```

By default the sync script looks for:

- `~/Downloads/GoogleService-Info.plist`
- `~/Downloads/google-services.json`

Those paths are persisted in the ignored file `apps/build-harness-expo/.build-harness.local.json`.

## Notes

- This harness targets Expo development builds, not Expo Go.
- Native iOS and Android folders are generated by `expo prebuild --clean` and remain ignored.
- The Expo config uses RNFB config plugins plus `expo-build-properties` with `useFrameworks: "static"` on iOS.
- The smoke screen confirms the JavaScript layer resolves RNFB modules, but the real validation path is prebuild plus native run.
