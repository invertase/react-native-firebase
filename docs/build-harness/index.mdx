The repository includes a checked-in bare React Native app at `apps/build-harness/` for manual build verification and Firebase setup testing.

Unlike `tests/`, which is the CI and Detox harness, the build harness is designed for local setup work:

- validating that workspace `@react-native-firebase/*` packages build and autolink correctly
- testing published RNFB package versions without changing the main test harness
- overriding Firebase iOS and Android SDK versions with the same supported hooks documented for consumer apps
- reusing local `GoogleService-Info.plist` and `google-services.json` files without committing them

## Quick start

From the repository root:

```bash
yarn app:sync
yarn app:start
yarn app:ios
```

Or build for Android:

```bash
yarn app:android
```

By default the sync script looks for:

- `~/Downloads/GoogleService-Info.plist`
- `~/Downloads/google-services.json`

You can override those paths at any time:

```bash
bash ./scripts/sync-build-harness.sh sync \
  --ios-google-services "$HOME/Downloads/GoogleService-Info.plist" \
  --android-google-services "$HOME/Downloads/google-services.json"
```

## Version overrides

The harness reads its defaults from `packages/app/package.json` for:

- Firebase JS SDK
- Firebase iOS SDK
- Android Firebase BOM
- Google Services, Crashlytics, Performance, and App Distribution Gradle plugin versions

Override any of them through the sync script:

```bash
bash ./scripts/sync-build-harness.sh sync \
  --firebase-ios 12.13.0 \
  --firebase-android-bom 34.13.0 \
  --google-services-gradle 4.4.4
```

The iOS override is applied through `FIREBASE_SDK_VERSION` / `RNFB_FIREBASE_IOS_SDK`, and the Android override is applied through `project.ext.'react-native'.versions.firebase.bom`.

## Workspace vs published RNFB packages

The harness defaults to local workspace packages so you can validate in-repo changes:

```bash
bash ./scripts/sync-build-harness.sh sync --rnfb-source workspace
```

To compare against a published release:

```bash
bash ./scripts/sync-build-harness.sh sync \
  --rnfb-source published \
  --rnfb-version 24.0.0
```

The current local setup is persisted in the ignored file `apps/build-harness/.build-harness.local.json`, so repeated `app:ios` and `app:android` runs reuse the last synced configuration.
